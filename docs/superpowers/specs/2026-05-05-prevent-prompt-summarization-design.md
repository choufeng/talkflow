# 防止提示词导致 LLM 过度总结

**日期**: 2026-05-05  
**状态**: 已批准

## 问题

当前润色提示词中「口语润色」段包含"去除重复表达、调整语序使其更通顺"，给了 LLM 过大自由度，导致模型频繁对输入进行大规模总结（Summarization），输出与原文偏差较大。

## 目标

修改提示词，确保输出尽可能贴近原文：

- 润色：仅去语气词 + 修错别字 + 去除口吃重复
- 翻译：逐句翻译，不总结、不删减

## 方案

选用 **白名单 + 黑名单** 策略（操作白名单限定 LLM 允许的行为范围，黑名单兜底禁止越界）。

### 润色提示词

```
你只被允许执行以下三项操作：

1. 删除无意义语气词和填充词
2. 修正明确的错别字和同音错误
3. 去除口吃和即时重复

以上三项之外，你被严格禁止：
- 禁止总结、概括、删减任何信息
- 禁止改写、变换或重组句式
- 禁止合并或拆分句子
- 禁止添加原文没有的内容
- 禁止将口语转换为书面语
- 输出必须保留原文的全部信息、表达顺序和口语特征
- 仅输出处理后的文本，不输出任何解释或说明
```

### 翻译提示词

增加"逐句翻译"和"禁止总结/删减/合并/拆分"约束。

## 实现

修改 `TalkFlow/Utils/PromptConfig.swift` 中的两个函数：
- `makePolishingSystemPrompt()` → 重写为白名单 + 禁令模式
- `makeTranslationSystemPrompt(language:)` → 追加逐句翻译和反总结约束
